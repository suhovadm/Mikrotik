ВАЖНЫЙ МОМЕНТ!
Если закинуть .exe-шник на virustotal.com, он определит 5 угроз:

DeepInstinct: MALICIOUS
SecureAge: Malicious
SkyHigh (SWG): BehavesLike.Win64.Fareit.vc
Sangfor Engine Zero: Trojan.Win32.Save.a
SentinelOne (StaticML): Static AI - Suspicious PE

Почему так?
Это ложные срабатывания, так называемые false positive.
Основные причины - это:
1) Сетевой функционал + создание и сохранение файлов = подозрительное поведение.
Открытие UDP-порта и запись файлов на рабочий стол. Антивирусы трактуют такое поведение как бэкдор или троян.
2) Использование системного 514 порта требует прав администратора.
Антивирусы настороженно относятся к программам, работающим с системными портами.
3) exe-шник не имеет цифровой подписи. Тут всё понятно.
Ну и + видимо есть какие-то вирусы, которые используют те же самые библиотеки Питона, поэтому virustotal решил, что эти куски кода это вирус...

Как пользоваться?

1) Открываем Winbox.

2)
Заходим в System - Logging - Actions - Remote
В Remote Address: прописываем IP своего компа
Remote Port: 514
Src. Address: 0.0.0.0
[ ] BSD Syslog

3)
Во вкладке Rules добавляем:
error - remote
info - remote
warning - remote

4)
Если 514 порт закрыт - открываем cmd от админа и пишем туда вот такую команду:
netsh advfirewall firewall add rule name="Syslog UDP 514" dir=in action=allow protocol=UDP localport=514
